# Quick Start Guide - دليل البدء السريع

## ما الذي تم تطويره؟

تم تطوير **3 ميزات رئيسية** للشات بوت:

### 1. فحص حالة الطلب المباشر ⚡
- عندما يسأل العميل "يمته يوصل طلبي؟"
- النظام يفحص قاعدة البيانات **مباشرة**
- يرد بمعلومات دقيقة 100% عن الطلب
- **بدون** الذهاب للـ AI

### 2. كشف الشكاوى والطلبات الخاصة 🔔
- النظام يكتشف تلقائياً:
  - الشكاوى (تالف، مكسور، ما وصلني)
  - الطلبات الخاصة (تخصيص، طباعة اسمي)
  - أسئلة الدفع (فيزا، زين كاش)
  - طلبات الجملة (100+ قطعة)
- يرسل **إشعار فوري** للموظف
- يرد للعميل برسالة انتظار مهذبة

### 3. نظام إشعارات ذكي 📬
- عند النقر على الإشعار → ينتقل للمحادثة
- يحتوي على رسالة العميل الأصلية
- سهل التتبع والرد

---

## كيف تعمل الميزات؟

### مثال 1: سؤال عن الطلب
```
العميل: "يمته يوصل طلبي؟"

النظام:
1. يكتشف أنه سؤال عن الطلب
2. يفحص قاعدة البيانات
3. يرد مباشرة:
   "طلبك #ORD-001 قيد التجهيز 📦
    راح يوصلك خلال 1-2 يوم"
```

### مثال 2: شكوى
```
العميل: "الطلب وصلني تالف"

النظام:
1. يكتشف كلمة "تالف" = شكوى
2. يرسل إشعار للأدمن:
   "المحادثة مع زيد تحتاج إلى موظف"
3. يرد للعميل:
   "لحظات أتأكد من سياسة الاسترجاع... 📋"
4. الأدمن ينقر الإشعار → يرد على العميل
```

---

## الكلمات المفتاحية

### أسئلة الطلب (فحص مباشر):
- وين طلبي
- يمته يوصل
- متى يوصل
- شنو تفاصيل طلبي
- حالة طلبي

### الشكاوى (إشعار للأدمن):
- شكوى
- مشكلة
- تالف
- مكسور
- ما وصلني
- اريد استرجاع

### الطلبات الخاصة (إشعار للأدمن):
- تخصيص
- تصميم خاص
- طباعة اسمي
- اضافة صورة

---

## الاختبار السريع

### 1. اختبر سؤال عن الطلب:
```
أرسل رسالة: "يمته يوصل طلبي؟"
المتوقع: رد مباشر من قاعدة البيانات
```

### 2. اختبر شكوى:
```
أرسل رسالة: "الطلب وصلني تالف"
المتوقع: 
- إشعار في لوحة التحكم
- رد انتظار للعميل
```

### 3. اختبر طلب خاص:
```
أرسل رسالة: "ممكن تطبع اسمي على القميص؟"
المتوقع:
- إشعار للأدمن
- رد انتظار
```

## أين تجد الإشعارات؟

### في قاعدة البيانات:
```sql
SELECT * FROM notifications 
WHERE type = 'human_agent_needed' 
ORDER BY created_at DESC;
```

### في الكود:
- جدول: `notifications`
- النوع: `human_agent_needed`
- البيانات: `conversation_id`, `lead_id`, `customer_message`
